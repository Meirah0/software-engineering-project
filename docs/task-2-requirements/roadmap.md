# Roadmap – Sukoon

## Overview
This roadmap describes the first development phases of the Sukoon smart hospital support application.

---

## Phase 1 – Project Foundation
### Goal
Define the project mission, target users, main requirements, and first feature priorities.

### Outputs
- Mission document
- Roadmap document
- Requirements table
- First feature specifications

---

## Phase 2 – Smart Hospital Navigation
### Goal
Allow users to find hospital departments and receive simple navigation guidance.

### Main Features
- Department search
- Step-by-step direction text
- Simple hospital map concept
- Clear navigation instructions

### Validation
Users should be able to find a department quickly and understand where to go.

---

## Phase 3 – Appointment and Reminder Support
### Goal
Help users manage appointment information and receive reminders.

### Main Features
- Add appointment details
- Reminder notifications
- Appointment overview
- Status tracking

### Validation
Users should receive reminders successfully and reduce the risk of missing appointments.

---

## Phase 4 – AI Emotional Support Chatbot
### Goal
Provide simple emotional support for users who feel stressed, anxious, or lost in the hospital.

### Main Features
- Calm chatbot responses
- Stress-reduction guidance
- Supportive messages
- Escalation recommendation if urgent help is needed

### Validation
Users should feel supported and understand that the chatbot is not a replacement for professional medical help.

---

## Phase 5 – Improved Dashboard and Analytics
### Goal
Create a project dashboard for requirements, progress, and future development tracking.

### Main Features
- Requirements status overview
- Progress indicators
- Feature priority tracking
- Iteration overview

---

## Future Ideas
- Multi-language support
- Indoor map integration
- QR-code-based hospital navigation
- Integration with hospital appointment systems
- Accessibility mode for elderly users
