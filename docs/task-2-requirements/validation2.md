# Validation 2 – Appointment and Reminder Support

## Related Feature
Feature 2 – Appointment and Reminder Support

## Validation Goal
To check whether users can add appointments and understand their reminders.

## Test Scenario
A user enters an appointment with date, time, department, and optional notes. The system displays the appointment and reminder information.

## Validation Method
- Functional test
- Usability test
- User feedback form

## Success Criteria
- The user can add an appointment without help.
- The appointment appears correctly in the appointment overview.
- The reminder information is visible and understandable.
- The user can edit or delete the appointment.
- At least 80% of users say the reminder feature is useful.

## Questions for Users
1. Was the appointment form easy to complete?
2. Was the reminder information clear?
3. Would this feature help you avoid missing appointments?
4. Was any information missing?
5. What would make the reminder more helpful?

## Expected Result
The reminder feature should help users feel more organized and reduce the risk of missing hospital appointments.
