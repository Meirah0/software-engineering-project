# Feature 2 – Appointment and Reminder Support

## Feature Title
Appointment and Reminder Support

## Iteration
Iteration 2

## Goal
The goal of this feature is to help users manage appointment information and receive reminders before their hospital visit.

## User Story
As a patient, I want to add my appointment details and receive a reminder so that I do not forget my hospital appointment.

## Functional Requirements
- The user can add appointment details.
- The user can enter date, time, department, and notes.
- The system stores the appointment information.
- The system shows upcoming appointments.
- The system sends or displays reminder notifications.

## Non-Functional Requirements
- The appointment form should be simple and short.
- The reminder information should be clear and visible.
- The system should avoid overwhelming the user with too many fields.
- Appointment data should be treated as sensitive information.

## Requirement Attributes
| Attribute | Description |
|---|---|
| Priority | High |
| Status | Planned |
| Owner | Meirah |
| Dependency | Reminder logic and appointment data storage |
| Risk | Users may enter incorrect appointment information |

## Acceptance Criteria
- The user can create an appointment entry.
- The appointment appears in the upcoming appointment list.
- The user can see the appointment date and time clearly.
- The reminder appears before the appointment.
- The user can edit or delete an appointment.
