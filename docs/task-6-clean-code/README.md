
# Task 6 – Clean Code Development

## Goal

The goal of this task is to show clean code principles in the project code and explain why the code is understandable, maintainable, and easy to improve.

This task includes:

1. Five clean code examples from the project code
2. Short explanations of why these examples are clean
3. A personal Clean Code cheat sheet with more than ten points

## Code Used

The clean code examples are based on the small Python pet project in the repository:

```text
mood_tracker.py
```

The project is simple, but it still shows important clean code ideas such as meaningful naming, readable structure, simple logic, user input, and clear output.

## Clean Code Examples

## 1. Meaningful File Name

The file name `mood_tracker.py` clearly describes the purpose of the program. This makes it easier to understand what the file is about before opening it.

## 2. Clear Variable Names

The code uses understandable variable names instead of unclear short names. Clear names make the code easier to read and reduce confusion.

Example:

```python
mood
```

The variable name is directly connected to the purpose of the program.

## 3. Simple Program Structure

The program follows a simple structure:

1. The user enters information
2. The program processes the input
3. The program gives an output

This makes the flow of the program easy to understand.

## 4. Readable Output

The output messages are written in a clear way for the user. Clean code is not only about the internal structure, but also about understandable communication with the user.

## 5. Avoiding Unnecessary Complexity

The program keeps the logic simple. It does not use complicated structures where simple code is enough. This makes the project easier to maintain and explain.

## 6. Easy to Extend

The project can be extended later with more moods, recommendations, or tracking features. A simple structure makes future improvements easier.

## Personal Clean Code Cheat Sheet

1. Use meaningful names for files, variables, and functions.
2. Keep code simple and readable.
3. Avoid unnecessary complexity.
4. Avoid duplicated code.
5. Use clear output messages.
6. Validate user input where needed.
7. Keep one responsibility per function or code section.
8. Use comments only when they add useful explanation.
9. Format code consistently.
10. Make the code understandable for another person.
11. Use readable error messages.
12. Test important behavior.
13. Refactor code when it becomes difficult to understand.
14. Keep project files organized.
15. Prefer clarity over clever solutions.

## Reflection

This task helped me understand that clean code is not only about making code work. It is also about making the code readable, maintainable, and easier to improve later. Even in a small Python project, clean code principles are useful because they make the project easier to explain and extend.
